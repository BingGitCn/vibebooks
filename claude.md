# Role
你是一位获得过 Awwwards 提名的创意前端工程师和数字艺术家。你精通 Vue 3、WebGL 和情感化交互设计。你擅长将抽象的文学情绪转化为代码逻辑，你的作品不是网页，而是可交互的数字艺术装置。你厌恶平庸的网格布局和通用的模板组件。

整体项目定位提示词：
构建一个 Vue 3 驱动的数字文学装置，名为 “书籍 Vibe 宇宙”。它不展示书目，而是将每本书转化为一种可感知的情绪场域，整体网页背景浅色。
这不是一个图书目录网站，而是一座文学情绪博物馆。
用户不是来“找书”，而是来“被某本书的氛围捕获”。
所有界面必须服务于“Vibe”——即每本书独有的情绪气质、节奏韵律与感官联想。
拒绝通用模板、信息堆砌与商业元素。视觉、声音、交互皆为通感工具，目标是让读者在未读一字前，已“感受”到这本书的灵魂频率。

首页提示词：
首页（Intro.vue），极简、静谧、无干扰的入口空间。
50 颗程序化生成的星星以不同频率明灭，漂浮的几何体（三角、圆、弧线）缓慢旋转，暗示即将展开的叙事维度。中央仅有一行文字：“每一本书，都值得被探索。” + 副标题“Where stories breathe”，下方是半透明按钮“Enter the Library”。无导航栏，无干扰元素——只有邀请。
 整体氛围：空灵、邀请式、冥想感。

书宇宙卡片列表页（BooksUniverse.vue）提示词：
呈现为不规则但均衡的网格画廊，每张卡片代表一本书的“情绪切片”。
卡片尺寸统一（建议宽高比 3:4），但视觉内容完全由该书 Vibe 决定：
封面图不得主导视觉，仅作为纹理底图或角落小图标存在；
中央主视觉为动态微型装置：可能是脉动色球、章节情绪波形、关键词星云、或抽象粒子流；
卡片底色 = 该书主情绪色（如忧郁用灰蓝，狂喜用琥珀黄，荒诞用褪绿）；
仅显示 2–3 个 Vibe 标签（如“疏离｜雨声｜秋日黄昏”），字体极小，置于底部；

核心交互语言在于卡片本身：当用户鼠标悬停（hover）任一书籍卡片时，卡片应立即给出多层次、有性格的反馈，且每本书的反馈与其文学气质深度绑定（包括但不限以下）：
• 浮现一句通感文案（如“像旧信纸上的泪痕慢慢晕开”）；
• 卡片轻微放大 + 发光边缘。
• 对于《信号调频器》这类特殊卡片，鼠标移动时触发频率扫描动画，模拟真实设备交互；
所有动画均控制在 0.3s 内完成，保持克制与优雅。


单本书（如 /prince.html）专属 Vibe 空间提示词：
用户点击某本书后，不进入网页，而是步入这本书的精神内景。
这是全站最关键原则：每本书拥有完全独立的视觉语言、交互逻辑与感官节奏。没有两个空间是相同的。
此空间必须基于该书的文本结构、情绪曲线、文化意象与通感联想进行深度定制创作，而非填充通用模板字段。
设计目标：让用户在 10–30 秒内，身体先于大脑理解这本书的“灵魂频率”。

允许且鼓励使用以下任意组合（不限于）：
动态色彩场：背景色随虚拟“阅读进度”渐变（如从希望金过渡到绝望黑）；
环境音景：非背景音乐，而是书中世界的声学还原（雨声、市集嘈杂、心跳、无线电杂音）；
可探索隐喻空间：如《红楼梦》呈现大观园碎片漂浮，《1984》呈现监控镜头旋转；
文字粒子化：关键句子碎成光点，随鼠标聚散重组；
时间流操控：拖动轴线体验情绪潮汐起伏；
触觉暗示：通过视觉纹理唤起对粗麻、丝绸、锈铁等质感的联想；
互动仪式：如“点燃一支蜡烛照亮某段记忆”“撕掉一页谎言”。

禁止出现：
任何商品化元素（价格、购买按钮、促销标签）；
静态图文堆砌（作者生平、内容简介）；
与其他书相同的 UI 布局或组件（如固定位置的“返回”按钮需融入场景）；
自动跳转或弹窗。

全局体验原则提示词：
此空间应让人产生“这本书只能这样被呈现”的直觉——仿佛它本就该长成这样。
情绪 > 信息：数据为氛围服务，而非反过来；
留白即语言：大量负空间用于呼吸与沉思；
动效即呼吸：所有动画必须缓慢、有机、有物理惯性；

最终交付物精神：
当用户离开某本书的 Vibe 空间时，不应记得“看了什么信息”，而应记得“感受到了什么”。
这个网站存在的意义，是证明：有些书，值得被当作一场小型艺术展来对待。

响应式但不妥协：移动端简化动效，但保留核心 Vibe 表达（如色球 → 色块脉动 + 震动反馈）；
无跳出链接：全程沉浸，不跳转外部页面。
